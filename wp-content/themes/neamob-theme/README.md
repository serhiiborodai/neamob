## Структура главной страницы

Главная страница состоит из следующих секций (сверху вниз):

1. **Hero Section** — главный баннер с заголовком
2. **Logo Slider** — бегущая строка логотипов партнёров
3. **What We Do** — аккордеон с услугами
4. **Value Section** — блок с ноутбуком и статистикой
5. **Case Studies** — карточки кейсов
6. **Testimonials** — слайдер отзывов
7. **CTA Section** — призыв к действию

---

## Как редактировать контент

### 1. Hero Section (главный баннер)

**Где:** Страницы → Home (или главная) → прокрутить вниз до блока "Hero Section"

| Поле | Описание |
|------|----------|
| Title | Заголовок. Для переноса строки используй `<br>` |
| Description | Текст под заголовком |
| Button Text | Текст кнопки (например "Let's Chat") |
| Button Link | Ссылка кнопки |

---

### 2. Логотипы партнёров (Logo Slider)

**Где:** В меню → **Partner Logos**

**Как добавить:**
1. Partner Logos → Add New
2. Заполнить название (для внутреннего использования)
3. Загрузить логотип (поле "Logo")
4. Указать порядок отображения (поле "Display Order")
5. Опубликовать

**Требования к логотипам:**
- Формат: SVG или PNG с прозрачным фоном
- Цвет: светлый/белый (отображается на тёмном фоне)
- Высота: ~40px (масштабируется автоматически)

---

### 3. Услуги / What We Do (аккордеон)

**Где:** В меню → **Services**

**Как добавить/редактировать:**
1. Services → Add New (или редактировать существующий)
2. Заполнить:
   - **Title** — название услуги
   - **Short Description** — текст для аккордеона на главной
   - **Content** — полный текст для страницы услуги
3. Опубликовать

**Порядок отображения:** Задаётся через "Order" в правой колонке редактора

---

### 4. Value Section (блок с ноутбуком)

**Где:** Страницы → Home → прокрутить вниз до блока "Value Section"

| Поле | Описание |
|------|----------|
| Title | Заголовок секции |
| Description | Описание под заголовком |
| Button Text | Текст кнопки |
| Button Link | Ссылка кнопки |
| Stats Card Text | Текст в синей карточке |
| Stats Number | Число (например "+358%") |
| Stats Label | Подпись под числом |
| Laptop/Dashboard Image | Изображение ноутбука |
| Report Tags | Теги внизу секции (Repeater) |

**Теги (Report Tags):**
- Нажми "Add Tag" для добавления
- Каждый тег имеет: текст, стиль (outline/icon/icon-alt), флаг "только иконка"

---

### 5. Case Studies (кейсы)

**Где:** В меню → **Case Studies**

**Как добавить:**
1. Case Studies → Add New
2. Заполнить:
   - **Title** — внутреннее название
   - **Client Name** — название клиента (отображается)
   - **Client Logo** — логотип клиента
   - **Badge Value** — значение бейджа (например "+32%")
   - **Badge Text** — текст бейджа (например "REVENUE")
   - **Excerpt** — текст для карточки на главной
   - **Content** — полный текст кейса
3. Опубликовать

**На главной отображаются:** 2 последних кейса

**Архив всех кейсов:** `/case-studies/`

---

### 6. Отзывы (Testimonials)

**Где:** В меню → **Testimonials**

**Как добавить:**
1. Testimonials → Add New
2. Заполнить:
   - **Title** — для внутреннего использования
   - **Quote Text** — текст цитаты
   - **Author Name** — имя автора
   - **Author Position** — должность
   - **Company Logo** — логотип компании (SVG/PNG, белый цвет)
   - **Author Photo** — фото автора (опционально)
3. Опубликовать

**Порядок слайдов:** Задаётся через "Order" в редакторе

---

- **Текст:** DM Sans (Regular, Medium, 400-600)

| Название | HEX | Использование |

| Primary | `#0094FF` | Кнопки, акценты |

| Text Dark | `#000A18` | Заголовки |
| Text | `#495B75` | Основной текст |

## SCSS Компиляция (для разработчиков)

```bash
cd wp-content/themes/neamob-theme

# Установить зависимости
npm install

# Скомпилировать один раз
npm run build

# Следить за изменениями
npm run watch
```

**Структура SCSS:**
```
scss/
├── style.scss          # Главный файл (импорты)
├── base/               # Переменные, миксины, типографика
├── layout/             # Header, Footer, Grid
├── components/         # Кнопки, карточки, слайдеры
└── blocks/             # Gutenberg блоки
```

## Защита от индексации

Сайт закрыт от поисковых роботов:
- `robots.txt` — запрещает индексацию
- `.htaccess` — X-Robots-Tag: noindex
- WordPress настройка — blog_public = 0

**Чтобы открыть для индексации:**
1. Удалить или изменить `robots.txt`
2. Убрать X-Robots-Tag из `.htaccess`
3. Настройки → Чтение → убрать галочку "Попросить поисковые системы не индексировать"
