# Гид по редактированию контента

## Суть CMS

**CMS = контент управляется из админки, без правки кода.** Пользователь заходит в WordPress, меняет текст/картинки — сайт обновляется. Никакого PHP, git, деплоя.

---

## Текущая карта контента

### ✅ Где редактировать

| Контент | Где редактировать | Путь в админке |
|---------|-------------------|----------------|
| **Меню** (Services, About, подпункты и т.д.) | Внешний вид → Меню | Заголовок пункта = текст в меню, URL = куда ведёт ссылка |
| **Главная** (Hero, Value, и т.д.) | Редактировать страницу Home | Контент → Страницы → Home → блоки ACF справа |
| **Theme Settings** (кнопка Let's Chat, контактная форма, footer) | Theme Settings | Слева в сайдбаре админки |
| **Страницы сервисов** (Growth Strategy, Data Analytics…) | Контент → Страницы → выбери страницу | У каждой страницы — ACF: Hero, Overview, What we do, What sets us apart |
| **Case Studies, Jobs, Team, FAQ** | Соответствующие пункты меню | Case Studies, Jobs, Team Members, FAQs |
| **Блог** | Записи | Записи → Добавить / Все записи |

### ⚠️ Проблема: Services — два источника

Сейчас сервисы живут в **двух местах**:

1. **Services (CPT)** — Контент → Services  
   - Используется на **главной** в блоке "What We Do" (аккордеон)  
   - Ссылка "Learn More" ведёт на `/service-items/xxx/`  
   - Редактируется: заголовок, краткое описание (Short Description)

2. **Страницы** (Growth Strategy, Data Analytics…) — Контент → Страницы  
   - Используются в **меню** как подпункты Services  
   - Отдельный контент: Hero, Overview, What we do, What sets us apart  
   - URL: `/growth-strategy-planning/`, `/data-analytics-insights/` и т.д.

**Следствие:** меню ведёт на страницы, аккордеон на главной — на другой тип записей. Разные названия, разный контент, двойное ведение.

---

## Что должно быть

**Один источник для сервисов.** Два варианта:

### Вариант A: только страницы (проще для пользователя)

- Убрать CPT Services
- На главной аккордеон строится из дочерних страниц Services
- Меню и главная ссылаются на одни и те же страницы
- Редактирование: Контент → Страницы → Services → дочерние страницы

### Вариант B: только Service CPT

- Убрать отдельные страницы сервисов
- Шаблон `page-service.php` переименовать в `single-service.php`
- Меню и главная ссылаются на записи Services
- Редактирование: Контент → Services

---

## Что редактируется в меню (подпункты)

В **Внешний вид → Меню** каждый пункт — это:

- **Заголовок** — что видно в меню (можно менять независимо от страницы)
- **URL** — куда ведёт клик (страница, запись, произвольная ссылка)

Родительский пункт «Services» с `#` — это заголовок без ссылки. Подпункты — его дочерние пункты. Редактируешь заголовок подпункта — меняется текст в меню. Редактируешь привязанную страницу — меняется страница, на которую ведёт ссылка (если пункт привязан к странице).

---

## Рекомендация

Свести сервисы к одному источнику (лучше Вариант A — страницы), чтобы не было двух параллельных сущностей. После этого в документе можно будет написать однозначно: «Сервисы редактируются здесь: …».
